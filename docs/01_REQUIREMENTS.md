# 01 要件定義

## 背景

ブックマークは業務、学習、調査、購入検討、開発資料など複数の文脈で増え続ける。既存の Chrome ブックマーク管理は保存と階層整理には向くが、後から「ドメイン」「サイト目的」「用途」「テーマ」のような観点で整理し直すには手作業が多い。

## 解決したい課題

- 未整理のブックマークが増え、後から再利用しにくい。
- ドメイン単位だけでは、同じサイト内の用途やテーマの違いを表現しにくい。
- 既存フォルダを壊さず整理し直すには、変更前の確認と復元手段が必要である。
- 自動分類の根拠が見えないと、ユーザーが安心して一括変更できない。

## 対象ユーザー

- Chrome のブックマークを業務や学習で大量に使うユーザー。
- 100 件以上のブックマークを整理したいが、一括移動には不安があるユーザー。
- 自分の分類ルールを編集しながら、継続的に整理品質を上げたいユーザー。

## MVP 機能要件

| ID | 要件 | 受け入れ条件 |
| --- | --- | --- |
| FR-1 | 既存ブックマークツリーを読み込む | URL を持つブックマークを一覧化できる |
| FR-2 | 複数軸の分類候補を提示する | ドメイン、目的、用途、テーマの値を結果に含める |
| FR-3 | 分類理由を表示する | 適用されたルール名またはフォールバック理由を表示する |
| FR-4 | 移動先フォルダをプレビューする | 実行前に `分類済みブックマーク / 大項目 / 中項目` の作成予定フォルダと移動予定ブックマークを確認できる |
| FR-5 | ルールを編集する | Options 画面で JSON 形式のルールを編集でき、有効な JSON ルールは分類先設定、サイト割り当て、ルール管理にも反映される |
| FR-6 | 整理を適用する | 確認後に必要なフォルダを作り、対象ブックマークを移動できる |
| FR-7 | 直前復元を提供する | 保存済みスナップショットから元の親フォルダへ戻せる |
| FR-8 | テスト用バックアップを作成する | 手動テスト前のブックマーク配置を保存できる |
| FR-9 | バックアップから復元する | 保存時点の親フォルダと順序へブックマークを戻せる |
| FR-10 | 分類先を設定する | 既存または新規の大項目、中項目、アイコン、表示名、優先度、軸を設定でき、URL なしでは分類先ルール、URL ありでは同じ分類先へのドメインルールを生成できる |
| FR-11 | 現在のブックマークからルール候補を作成する | ローカルのブックマークツリーを解析し、頻出ドメインと内容から候補ルールを提示できる |
| FR-12 | ルールをグラフィカルに確認する | JSON だけでなく、分類先ごとのグループ、グリッド、リスト表示でルールとアイコンを確認できる |
| FR-13 | リンク切れ候補を検出する | 整理時に HTTP/HTTPS ブックマークを確認し、HTTP 404/410 の候補を削除候補として表示できる |
| FR-14 | リンク切れ候補を確認後に削除する | 削除候補一覧と確認ダイアログを経由した場合だけ、該当ブックマークを削除できる |
| FR-15 | サイトを分類先へ割り当てる | 現在のブックマークをホスト名とドメイン名で一覧化し、左側の大項目・中項目へドラッグ & ドロップするとドメインルールを作成または更新できる |
| FR-16 | サイト割り当て対象を絞り込む | 分類済みサイトを非表示にでき、サイトカードには favicon サムネイルが表示され、長すぎるサイト名やサンプル名はカード内で省略表示される |
| FR-17 | 主要サイトから候補ルールを作成する | 現在のブックマーク以外に、主要検索、ポータル、業務、開発、ショッピング、ニュースなどの代表的サイトから候補ルールを作成できる |
| FR-18 | 候補作成の軸を選択する | 目的、用途、テーマの対象軸を選んで候補ルールを作成できる |
| FR-19 | 空フォルダを削除する | 整理適用後に空になった元フォルダと整理済みブックマーク配下の空フォルダを削除できる |
| FR-20 | Side Panel 内でルールを編集する | Side Panel のルール編集ボタンから Options 相当の編集画面を表示でき、従来どおり別タブの Options も開ける |
| FR-21 | 全ルールを削除する | Options で確認ダイアログを経由して編集中の分類ルールを空にでき、保存すると空のルールセットを維持できる |
| FR-22 | 削除予定カテゴリで整理時削除する | `整理 / 削除予定` に分類されたブックマークは、整理プレビューと確認ダイアログを経由した場合だけ削除できる |

## 非機能要件

- 分類や整理のために外部 API へブックマークの完全 URL やタイトルを送信しない。favicon サムネイル表示時のみ、ドメイン名を外部 favicon サービスへ送信する場合がある。
- `bookmarks`、`storage`、`sidePanel` を基本権限とし、リンク切れチェックのために `http://*/*` と `https://*/*` の host permission を使う。
- 一括移動は必ずプレビューと確認を経由する。
- リンク切れ削除は HTTP 404/410 に限定し、通信エラー、認証エラー、アクセス制限、一時的なサーバーエラーは削除候補にしない。
- 削除予定カテゴリによる削除は、ユーザーが明示的に削除予定へ分類したブックマークだけを対象にし、整理適用時の確認を必須にする。
- 空フォルダ削除は整理適用時の確認ダイアログを経由し、Chrome のルートフォルダや整理済みブックマークのルートは削除対象にしない。
- バックアップ復元は、保存時点に存在したブックマーク本体の配置復元に限定し、未知の新規ブックマークや空フォルダ削除は自動で行わない。
- Chrome API に依存しない分類ロジックは Node.js で自動テストできるようにする。
- Markdown 文書は UTF-8 で保存し、文字化け断片を残さない。

## MVP 対象外

- クラウド同期。
- LLM や外部 API に依存する分類。
- Chrome 以外のブラウザへの完全対応。
- Chrome Web Store 公開用の最終アイコン、スクリーンショット、プライバシーポリシー完成。

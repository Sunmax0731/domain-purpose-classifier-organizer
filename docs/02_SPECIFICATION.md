# 02 仕様

## 画面

| 画面 | 役割 |
| --- | --- |
| Popup | 状態要約、Side Panel と Options への導線 |
| Side Panel | スキャン、分類結果一覧、Options の埋め込み編集、移動プレビュー、リンク切れチェック、リンク切れ削除、テスト用バックアップ、適用、直前復元 |
| Options | 分類先設定、サイト割り当て、現在ブックマークからの候補生成、分類ルールカード管理、分類ルール JSON の編集、初期ルール復元 |

Side Panel 上部のステータスは、スキャン、Options 埋め込み表示、リンク切れ確認、リンク切れ削除、バックアップ作成、バックアップ復元、整理適用、直前復元の進行中状態だけでなく、完了または失敗した状態も表示する。

## データモデル

### BookmarkItem

- `id`: Chrome bookmark id
- `title`: 表示名
- `url`: ブックマーク URL
- `parentId`: 現在の親フォルダ id
- `index`: 現在フォルダ内の順序
- `path`: 現在のフォルダパス
- `domain`: URL から抽出したドメイン

### ClassificationRule

- `id`
- `name`
- `type`: `domain`、`keyword`、`category` のいずれか。`category` は分類先メタデータ専用で、ブックマークには直接一致しない。
- `axis`: `purpose`、`usage`、`theme` のいずれか
- `priority`
- `pattern`
- `parentFolder`: 大項目。整理時の親フォルダ名
- `targetFolder`: 中項目。整理時の子フォルダ名
- `targetIcon`: 組み込みアイコン名、短いカスタム文字列、または画像 URL
- `reason`

### ClassificationResult

- `bookmarkId`
- `title`
- `url`
- `domain`
- `axes`: ドメイン、目的、用途、テーマの分類結果
- `suggestedCategory`
- `suggestedTarget`: `parentFolder`、`targetFolder`、`targetIcon`
- `targetPathSegments`: 整理先の大項目と中項目
- `suggestedFolder`
- `targetIcon`
- `confidence`
- `reasons`
- `matchedRuleIds`

### ReorganizationPlan

- `id`
- `createdAt`
- `rootTitle`
- `axis`
- `actions`: 整理予定。`type: move` は移動、`type: delete` は削除予定カテゴリによる削除
- `foldersToCreate`: 作成予定フォルダパス
- `rollbackSnapshot`: 復元用の親フォルダ、順序、タイトル、URL

### BookmarkBackup

- `id`
- `createdAt`
- `source`: `manual`
- `bookmarkCount`
- `snapshot`: 保存時点の URL ブックマーク ID、タイトル、URL、親フォルダ、順序、パス

### LinkCheckResult

- `bookmarkId`
- `title`
- `url`
- `domain`
- `path`
- `status`: `ok`、`broken`、`warning`、`skipped`
- `httpStatus`
- `reason`
- `deletionCandidate`: HTTP 404/410 の場合だけ `true`

## 主要フロー

1. Side Panel で「スキャン」を実行する。
2. `chrome.bookmarks.getTree` でブックマークツリーを取得する。
3. ルールを読み込み、URL を持つブックマークを分類する。
4. 選択軸に基づき、`rootTitle / parentFolder / targetFolder` の整理先フォルダ案を生成する。
5. ユーザーがプレビューを確認する。
6. 確認ダイアログで了承した場合だけ、フォルダ作成、移動、削除予定カテゴリのブックマーク削除を実行する。
7. 移動後に空になった元フォルダと、整理済みブックマーク配下の空フォルダを削除する。
8. 実行履歴、復元スナップショット、空フォルダ削除件数を `chrome.storage.local` に保存する。
9. 直前復元では保存済み `parentId` と `index` を使い、可能な範囲で元へ戻す。削除済みの元フォルダは保存済みパスから再解決または再作成する。

## テスト用バックアップフロー

1. Side Panel で「現在状態をバックアップ」を押す。
2. `chrome.bookmarks.getTree` で現行ブックマークツリーを取得する。
3. URL を持つブックマークの `id`、`parentId`、`index`、`title`、`url`、`path` を `chrome.storage.local` に保存する。
4. 繰り返しテスト後に「バックアップから復元」を押す。
5. 復元ジョブを作成し、20 件ずつ処理しながら進捗を UI に表示する。
6. 保存済みバックアップの各ブックマークを保存時点の `parentId` と `index` へ戻す。
7. `index` 指定で戻せない場合は、同じ親フォルダへ `index` なしで戻し、順序調整件数として表示する。
8. 保存済み `parentId` へ戻せない場合は、保存済み `path` から復元先フォルダを再解決または再作成して戻す。
9. 1 件の移動が一定時間で返らない場合は警告へ回し、復元処理全体を止めない。
10. 親フォルダが存在しないなど復元できない項目は警告件数と詳細として表示する。
11. 復元完了後は Side Panel 上部のステータスを `バックアップ復元完了: 復元数/総数` へ更新する。

バックアップ復元は、ブックマーク本体の配置復元を目的とする。テスト中に作成された空フォルダは自動削除しない。

## リンク切れチェックフロー

1. Side Panel で「リンク切れを確認」を押す。
2. `chrome.bookmarks.getTree` で現行ブックマークツリーを取得する。
3. HTTP/HTTPS URL だけを対象にし、Chrome 内部 URL、`file:`、`mailto:` などは `skipped` にする。
4. Service Worker が少数ずつリンクを確認し、Side Panel へ進捗を表示する。
5. 原則として `HEAD` で確認し、`HEAD` が未対応の場合だけ `GET` で再確認する。
6. HTTP 404/410 を `broken` かつ `deletionCandidate` として扱う。
7. 2xx/3xx は到達可能として扱う。
8. 401、403、405、429、5xx、通信エラー、タイムアウトは `warning` として表示し、削除候補にはしない。
9. 「確認してリンク切れを削除」を押し、確認ダイアログで了承した場合だけ、候補ブックマークを削除する。
10. 削除後は再スキャンし、整理プレビューから削除済みブックマークを外す。

削除済みブックマークは通常のバックアップ復元では再作成しない。削除操作は HTTP 404/410 に限定し、ユーザー確認を必須にする。

## Side Panel からのルール編集フロー

1. Side Panel の分類結果セクションには `Side Panelで編集` と `別タブ` の 2 つの導線を置く。
2. `Side Panelで編集` を押すと、Side Panel 内に Options 相当の編集画面を iframe で表示する。
3. iframe は初回表示時にだけ `options/options.html` を読み込み、閉じる操作では非表示にする。
4. `別タブ` は従来どおり `chrome.runtime.openOptionsPage()` を使い、Chrome の Options タブを開く。
5. Side Panel 内で編集したルールも Options タブと同じ `chrome.storage.local` を使うため、保存後の分類はどちらの導線でも同じ結果になる。

## 分類ルール

- ドメイン一致は `example.com` とサブドメインを対象にする。
- キーワード一致はタイトル、URL、ドメイン、パスを小文字化して判定する。
- 英数字だけのキーワードは単語または URL 区切り単位で一致させる。例えば `ai` は `AI tool` には一致するが、`Gmail` や `AirStation` の部分文字列には一致させない。
- 複数ルールが一致した場合は軸ごとに `priority` が高いルールを採用する。
- 既存ルールに `parentFolder` または `targetIcon` がない場合は、`targetFolder` から大項目とアイコンを補完する。
- 保存済みルールが古い場合でも、現在の初期ルールに含まれるドメインルールと汎用キーワードルールを補完する。同じ `id` または同じ `type`、`axis`、`pattern` のルールがある場合は保存済みルールを優先する。
- 選択軸に一致がない場合でも、他の軸に一致したルールがあれば最優先の一致ルールを整理先に使う。
- どの軸にも一致がない場合は `未分類 / ドメイン名` へフォールバックし、理由に「ドメインによる暫定分類」を表示する。
- `整理 / 削除予定` に分類されたブックマークは整理計画上で `delete` アクションになり、移動先フォルダは作成しない。

## 分類ルール編集フロー

1. Options は `分類先設定`、`サイト割り当て`、`ルール管理`、`JSON 編集` の 4 モードを持つ。
2. `分類先設定` では、大項目、中項目、アイコン、軸、優先度、表示名、理由を指定し、URL なしで `category` ルールとして分類先を追加できる。
3. `分類先設定` でサイト URL を任意入力した場合は、同じ分類先へ紐づく `domain` ルールも追加できる。URL は必須ではなく、分類先設定フォームの後半に配置する。
4. 大項目と中項目は、現在の有効な JSON ルールと候補作成で作られた未追加候補から生成した分類先のドロップダウンから選択できる。初期分類先プリセットは、明示的な初期ルール復元または候補作成の結果として扱い、空の JSON ルールには自動混入しない。`新規作成...` を選ぶと入力フォームを有効化し、新しい分類先を作成できる。
5. 分類先設定フォームは必須項目と任意項目をバッジで可視化する。大項目、中項目、アイコン、軸、優先度は必須、表示名、理由、サイト URL、認識ドメインは任意または自動項目として扱う。
6. 分類先設定では、追加済みの大項目・中項目をカード一覧で表示し、編集または削除できる。編集では同じ分類先に属する既存ルールの大項目、中項目、アイコンをまとめて更新し、削除では対象分類先に属するルールを確認後に削除する。
7. アイコンは組み込み候補から選択できる。`カスタム...` を選ぶと短い文字列または画像 URL を指定でき、ルールカードでは分類先アイコンまたはサムネイルとして表示する。
8. 候補作成では、候補元として `現在のブックマーク`、`主要サイト`、`現在 + 主要サイト` を選択できる。
9. `現在のブックマーク` では `chrome.bookmarks.getTree` を読み取り、頻出ドメイン、タイトル、URL、既存フォルダパスから候補分類先を推定する。
10. `主要サイト` では、検索、ポータル、地図、業務、開発、ショッピング、ニュース、プロフィールなどの代表的なドメインとキーワードから候補ルールを作成する。
11. 候補作成では、目的、用途、テーマの対象軸を選択できる。
12. 候補ルールは既存の同一 `type`、`axis`、`pattern` のルールを除外し、ユーザーが確認してから現在のルールへ追加する。
13. `サイト割り当て` では、左に現在の有効な JSON ルールと候補作成で作られた未追加候補から生成した大項目・中項目の分類先、右に現在のブックマークから作ったサイト一覧を表示する。JSON ルールと候補がどちらも空の場合、分類先リストは空状態を表示する。
14. サイト一覧は favicon サムネイル、ホスト名、ルール化するドメイン名、件数、代表タイトルを表示し、グリッドとリストを切り替えられる。
15. サイト割り当てでは分類済みサイトを非表示にできる。
16. サイト名、ドメイン名、代表タイトルが長すぎる場合は、カード内で省略表示し、元の文字列は tooltip で確認できる。
17. サイトを分類先へドラッグ & ドロップすると、該当ドメインの `purpose` ルールを作成する。既存の同一ドメインルールがある場合は分類先を更新する。
18. サイト割り当ては分類ルールを編集する機能であり、実際のブックマーク移動は Side Panel のスキャン、プレビュー、適用フローで行う。
19. `ルール管理` では、分類先ごとのグループ表示、全ルールのグリッド表示、コンパクトなリスト表示を切り替えられる。
20. ルール管理の削除は編集中のルール配列から対象ルールを除外する。初期ルールの自動補完は、ユーザーが保存した編集済みルールには再適用しない。
21. `全ルール削除` は確認ダイアログ後に編集中のルール配列を `[]` にし、候補もクリアする。永続化は `保存` を押した時点で行う。
22. `JSON 編集` では従来どおり JSON を直接編集できる。
23. JSON 編集欄の内容が有効なルール配列になった時点で、分類先設定の分類先ドロップダウン、サイト割り当ての分類先、サイト割り当ての分類済み判定、ルール管理、候補作成時の既存ルール判定へ未保存の編集状態として反映する。空配列 `[]` の場合、分類先設定とサイト割り当ては保存済みルール由来の分類先を表示せず、候補作成で作った未追加候補がある場合だけ候補由来の分類先を表示する。
24. JSON が不正な場合は古い有効な編集状態を維持し、JSON 編集以外のモードへ切り替えようとした時点でエラーを表示して切り替えを止める。
25. 分類先設定、サイト割り当て、JSON 操作はいずれも `保存` を押した時点で `chrome.storage.local` に反映する。
26. favicon サムネイルは外部 favicon サービスから取得し、取得できない場合はドメイン名から生成したローカルの頭文字タイルを表示する。

## 初期分類先

初期ルールと候補生成は、現在のブックマークを次の大項目と中項目へ寄せる。

| 大項目 | 中項目の例 | 主な対象 |
| --- | --- | --- |
| 開発 | コード管理、AI・自動化、ネットワーク | GitHub、AI ツール、Cloudflare、Tailscale |
| インターネット | 検索・ポータル、ニュース・情報、地図・交通 | Google、Yahoo!、Bing、ニュース、地図 |
| 業務 | 連絡・Office、プロジェクト管理 | Gmail、Office、Teams、Notion、Jira |
| メディア | SNS・投稿、動画・視聴 | X、Twitter、YouTube、Abema、動画 CDN |
| 生活 | 購入・金融、購入・価格比較、予約・手続き | Amazon、カード、決済、予約、申請 |
| 制作 | 画像・素材、UI・デザイン | PIXTA、画像素材、Material Design |
| サイト運営 | 分析、プロフィール | Google Analytics、Wix、ポートフォリオ |
| 知識 | 学習・リファレンス、調査・資料、調査・論文 | Docs、論文、Wiki、講座 |
| 娯楽 | イベント・ゲーム、ゲーム | ゲーム、音楽、イベント、Steam、Nintendo、PlayStation |
| 制限付き | 成人向け | 成人向け、R18、アダルト関連 |
| 整理 | 削除予定 | 削除予定、削除候補、不要ブックマーク、リンク切れとして明示したブックマーク |
| 未分類 | ドメイン別 | ルール未一致のドメイン |

## 例外処理

- Chrome API エラーは UI に表示する。
- 無効なルールは Options で保存しない。
- 移動前に対象件数が 0 件の場合は適用ボタンを無効化する。
- 復元時に移動元フォルダが存在しない場合は、その項目を警告として記録する。

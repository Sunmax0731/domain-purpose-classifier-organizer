- [x] Options のルール編集を分類先設定画面として構成する。

## フェーズ 6: 分類先管理 UI

- [x] 分類先を大項目と中項目に分け、整理先を親フォルダと子フォルダとして生成する。
- [x] 分類先設定で既存の大項目と中項目をドロップダウン選択できるようにする。
- [x] `新規作成...` 選択時に大項目または中項目の入力欄を有効化する。
- [x] 分類先ごとに組み込みアイコン、カスタム文字列、画像 URL サムネイルを設定できるようにする。
- [x] ルール管理をグループ、グリッド、リストで切り替え表示できるようにする。
- [x] 現在のブックマーク向けの初期ルールと候補生成を大項目と中項目の分類へ更新する。
- [x] `ai` などの短い英字キーワードを部分一致させず、誤分類を防ぐ。
- [x] 古い保存済みルールにも現在の初期ルールを補完し、既定ドメイン分類を維持する。
- [x] 選択軸に一致がない場合、他軸の一致ルールをドメインフォールバックより優先する。
- [x] 分類先管理 UI の静的テストと分類ロジックテストを追加する。

## フェーズ 7: リンク切れチェックと削除

- [x] HTTP/HTTPS ブックマークを少数ずつ確認するリンクチェックジョブを追加する。
- [x] HTTP 404/410 だけをリンク切れ削除候補として扱う。
- [x] 通信エラー、認証エラー、アクセス制限、一時的なサーバーエラーを削除候補から除外する。
- [x] Side Panel にリンク切れチェックの進捗、削除候補一覧、削除確認導線を追加する。
- [x] 削除後に再スキャンして整理プレビューを更新する。
- [x] リンクチェック用の host permission と権限説明を追加する。
- [x] リンク判定と manifest 権限の自動テストを追加する。

- [x] サイトを分類先へドロップすると、ドメインルールを追加または既存ルールを更新する。
- [x] サイト割り当ての静的テストとサイト一覧生成テストを追加する。

## フェーズ 9: ルール候補と整理後クリーンアップ

- [x] ルール管理の削除操作で初期ルールが即時復活しないよう、編集中ルールの正規化を初期補完なしに分離する。
- [x] 保存済みの編集済みルールはユーザー管理として扱い、削除した初期ルールを保存後に再補完しない。
- [x] サイト割り当てに分類済みサイトの非表示設定を追加する。
- [x] サイト割り当ての長いサイト名、ドメイン名、代表タイトルを省略表示する。
- [x] サイト割り当てのサイトカードに favicon サムネイルとフォールバック用の頭文字タイルを追加する。
- [x] JSON 編集で作成した有効なルールを分類先設定、サイト割り当て、ルール管理へ保存前に同期する。
- [x] 整理適用後に空になった元フォルダと整理済みブックマーク配下の空フォルダを削除する。
- [x] 現在のブックマーク以外に、主要サイトの静的ルールバンクから候補作成できるようにする。
- [x] 候補作成時に目的、用途、テーマの軸を選択できるようにする。
- [x] ルール削除、主要サイト候補、サイト割り当て表示制御、空フォルダ削除の自動テストを追加する。
- [x] Options に全ルール削除ボタンを追加する。
- [x] 初期ルールへゲーム、成人向け、削除予定の分類を追加する。
- [x] `整理 / 削除予定` の分類結果を整理適用時の削除アクションへ変換する。

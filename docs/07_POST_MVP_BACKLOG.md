# 07 MVP 後バックログ

## 方針

ここにある項目は、現在の MVP リリースを止めるものではない。安全な整理、復元、ルール編集、リンク切れ削除の基本機能が安定した後に、利用頻度とリスクを見て順に実装する。

## 優先候補

| 優先度 | 項目 | 理由 | 完了条件 |
| --- | --- | --- | --- |
| P1 | 分類結果の手動補正 UI | 個別ブックマークを見ながら例外ルールを作る需要が高い | Side Panel の結果から分類先を変更し、ルールまたは今回限りの例外として保存できる |
| P1 | ルールのエクスポート / インポート | 複数 Chrome プロファイルやバックアップ用途で必要 | JSON ファイルとして保存、読み込み、検証、差分確認ができる |
| P1 | Chrome Web Store 公開準備 | 配布に必要 | アイコン、スクリーンショット、説明文、プライバシーポリシーを整備する |
| P2 | 大量ブックマーク向け仮想スクロール | 1000 件以上で Side Panel と Options の描画負荷を抑える | 分類結果、変更プレビュー、サイト割り当て、ルール管理でスクロール時の体感遅延を抑える |
| P2 | ルール衝突の可視化 | 優先度や複数軸の競合を理解しやすくする | 同一ドメインや近いキーワードの競合候補を一覧し、優先度変更できる |
| P2 | アクセシビリティ確認 | キーボード操作と読み上げ対応を強化する | タブ移動、フォーカス表示、ボタン名、コントラストを確認し、主要操作をキーボードだけで実行できる |
| P3 | Chrome 拡張 E2E テスト | 実機手動確認の回帰負担を下げる | 専用プロファイルで読み込み、Side Panel、Options、保存、復元の主要操作を自動確認できる |
| P3 | 詳細なリンクチェック設定 | サイト負荷や誤判定を制御したい | タイムアウト、同時実行数、HEAD/GET 再試行、対象除外ドメインを設定できる |

## 今回実装しない理由

- MVP の中核は、ブックマーク整理を安全に試し、戻せることにある。
- 上記の多くは便利機能または配布準備であり、現在の分類、適用、復元の品質確認を複雑にする。
- とくに仮想スクロールや E2E テストは構成変更の影響が大きいため、リリース候補を固定してから別タスクで扱う。

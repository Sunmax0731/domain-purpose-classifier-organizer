# 01 要件定義: ブックマーク分類・フォルダ自動整理

## 背景

保存済みブックマークは増え続ける一方で、あとから探し直す操作や目的別に整理し直す操作に大きな手間がかかる。既存の管理手段では一覧性や検索性はあっても、複数の観点を切り替えながら実用的に分類し、フォルダ構造へ反映する用途には不足がある。

## 課題

- ブックマークが未整理のまま蓄積し、再利用しにくい。
- ドメイン単位だけでは用途やテーマの違いを表現しにくい。
- 既存フォルダを壊さずに整理し直すには事前確認と安全策が必要である。

## 対象ユーザー

- Chrome のブックマークを業務や学習で大量に使うユーザー
- 大量のブックマークをフォルダ整理したいが、一括整理では不安があるユーザー
- 複数観点の分類ルールを自分で調整しながら運用したいユーザー

## ユーザー価値

- ブックマークを複数軸で分類して、あとから見つけやすくする。
- 分類結果をプレビューし、安全を確認したうえでフォルダ反映できる。
- 手動補正やルール調整により、運用しながら分類品質を上げられる。

## MVP 機能要件

- FR-1: 既存ブックマークツリーを読み込める。
- FR-2: ドメイン、サイト目的、用途、テーマなどの候補分類を提示できる。
- FR-3: 分類結果ごとの移動先フォルダを一覧でプレビューできる。
- FR-4: ユーザーが分類ルールを編集し、再分類を実行できる。
- FR-5: 実行前後の差分を確認できる。
- FR-6: ロールバックまたは直前復元の仕組みを持つ。

## 非機能要件

- `chrome.bookmarks` API を使う前提で、権限説明を分かりやすく示す。
- 大量ブックマークでも一覧表示と分類プレビューが破綻しない。
- 危険操作は確認ダイアログとプレビューを必須にする。

## MVP の対象外

- クラウド同期を伴う高度な共有機能
- LLM や外部 API に依存した高コスト分類
- Chrome 以外のブラウザ完全対応

## 受け入れ条件

- 100 件以上のブックマークで分類候補が提示される。
- 分類プレビューに基づいてフォルダ構造の変更を確認できる。
- ルール編集とロールバックの主要フローを手動テストで確認できる。

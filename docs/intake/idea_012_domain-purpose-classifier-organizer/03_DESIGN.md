# 03 設計: ブックマーク分類・フォルダ自動整理

## 画面設計方針

- 一括変更前の不安を減らすため、常に差分プレビューを中心に配置する。
- 分類理由を短い説明で見せ、ルール調整との対応関係を理解しやすくする。
- 危険操作は色と文言で明確に区別する。

## UI セクション

- ヘッダー: スキャン実行、保存済みルール、最終更新
- 左カラム: フォルダツリー、除外条件、フィルタ
- 中央: ブックマーク一覧、分類候補、分類理由
- 右カラム: 変更プレビュー、作成フォルダ、ロールバック情報

## 状態設計

- Idle
- Scanning
- Reviewing
- Applying
- Completed
- Failed

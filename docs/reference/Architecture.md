# アーキテクチャガイド: ChromeExtension

## 目的

このガイドは、`domain-purpose-classifier-organizer` を ChromeExtension として実装する際の責務分割と設計判断を整理するためのものです。

## 想定構成

- `service worker`
  - ブックマーク取得
  - 分類ジョブ実行
  - 差分計算
  - ロールバック情報保存
- `Side Panel`
  - ブックマーク一覧
  - 分類候補表示
  - 差分プレビュー
  - 実行確認
- `Options`
  - ルール管理
  - 除外条件設定
  - バックアップ設定
- `storage.local`
  - 分類ルール
  - 実行履歴
  - 直前復元用スナップショット

## 責務分割

- 取得層
  - `chrome.bookmarks` から現在のツリーを読み込む。
- 判定層
  - ドメイン、タイトル、URL パス、テーマルールから分類候補を作る。
- 表示層
  - 分類候補、分類理由、移動先フォルダ、差分を分かりやすく出す。
- 実行層
  - フォルダ作成、移動、復元を安全に行う。

## 品質要件

- 可逆性: 実行前状態を保持し、直前復元できる。
- 説明可能性: なぜその分類になったかを表示できる。
- 操作性: 100 件以上のブックマークでもレビューしやすい。
- 審査適合性: 権限要求理由と単一目的を明確にする。

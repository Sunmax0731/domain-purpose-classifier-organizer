# Codex Workflow

## このアイデアでの作業順

1. ZIP 内の要件、仕様、設計を読む。
2. 今回触る範囲を `取得 / 分類 / プレビュー / 実行 / 復元` のどこかに限定する。
3. 変更前に、危険操作の有無と確認方法を明確にする。
4. 実装時は `service worker` と UI の責務を混ぜすぎない。
5. 検証時は、権限、分類候補、差分プレビュー、ロールバックを優先確認する。

## プロンプト解釈

- ユーザーの通常の指示は、そのまま内部で `目的 / 制約 / 期待出力 / 根拠 / 危険操作` に分解して扱う。
- 長文コンテキストでは、重要制約を先頭で確認し直す。
- ツール実行や複数ステップの前には短い preamble を出す。

## 開発時の優先順位

- まず安全性
- 次に分類の説明可能性
- 次に UI の分かりやすさ
- 最後に拡張性

# Content Script / Messaging

## 前提

このアイデアの主対象は Chrome のブックマークデータであり、MVP では `content script` を必須にしない。中心になるのは `chrome.bookmarks` と設定 UI である。

## 使う場合の位置づけ

- 現在閲覧中ページを一時分類対象として取り込む
- ページタイトルや見出しを補助情報として取得する
- サイト内の文脈を追加ヒントとして使う

## 基本方針

- ブックマーク整理の主要処理は `service worker` 側で持つ。
- `content script` は補助データ取得に限定する。
- メッセージは `request type` を明示し、戻り値の形を固定する。

## MVP で優先しないもの

- 常時注入
- 重い DOM 解析
- 外部通信を伴う複雑なページ評価

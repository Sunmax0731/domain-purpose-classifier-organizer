# 手動テスト項目と確認手順

## 前提

- 作業ディレクトリ: `D:\AI\ChromeExtension\domain-purpose-classifier-organizer`
- 読み込み対象: `D:\AI\ChromeExtension\domain-purpose-classifier-organizer\extension`
- Chrome は `chrome://extensions` でデベロッパーモードを有効にし、「パッケージ化されていない拡張機能を読み込む」から上記 `extension` フォルダを指定する。
- 実ブックマークで試す前に、Side Panel の `現在状態をバックアップ` を実行する。
- 整理適用、リンク切れ削除、バックアップ復元、直前復元は Chrome の確認ダイアログで内容を確認してから実行する。

## MT-0 事前確認

1. `npm test` を実行する。
2. すべてのテストが成功することを確認する。
3. `chrome://extensions` で拡張機能にエラーが表示されていないことを確認する。
4. 必要権限が `bookmarks`、`storage`、`sidePanel`、`http://*/*`、`https://*/*` であることを確認する。

期待結果:

- 自動テストが成功する。
- 拡張機能カードに未処理のエラーがない。
- リンク切れチェック用 host permission 以外の不要な権限が増えていない。

## MT-1 Popup と Side Panel 導線

1. Chrome ツールバーの拡張機能アイコンから Popup を開く。
2. `Side Panel を開く` を押す。
3. Side Panel が開き、上部ステータスが表示されることを確認する。
4. Popup の `Options を開く` を押し、Options タブが開くことを確認する。

期待結果:

- Popup から Side Panel と Options へ移動できる。
- Side Panel の初期状態でブックマーク件数、移動候補、作成予定が 0 として表示される。

## MT-2 Side Panel 内の Options 表示

1. Side Panel の分類結果セクションで `Side Panelで編集` を押す。
2. Side Panel 内に `分類ルール編集` セクションが表示され、Options 画面が埋め込み表示されることを確認する。
3. `閉じる` を押し、埋め込み表示が非表示になることを確認する。
4. `別タブ` を押し、従来どおり Options タブが開くことを確認する。

期待結果:

- Side Panel 内で Options が表示できる。
- 別タブ導線も残っている。
- 操作後に Side Panel 上部ステータスが変化する。

## MT-3 Options の分類先設定

1. Options の `分類先設定` タブを開く。
2. 大項目、中項目、アイコン、軸、優先度が必須バッジ付きで表示されることを確認する。
3. 表示名、理由、サイト URL、認識ドメインが任意または自動バッジ付きで表示されることを確認する。
4. サイト URL がフォーム後半にあり、未入力のままでも分類先を追加できることを確認する。
5. 各項目が「項目名の行」と「入力フォームの行」の 2 行で揃い、入力、選択、ボタン、認識済みドメイン表示の高さが揃うことを確認する。
6. JSON ルールが空で候補も未作成の場合、大項目と中項目は `新規作成...` のみ選べることを確認する。
7. 大項目と中項目で `新規作成...` を選び、必須項目を入力して `分類先を追加` を押す。
8. 分類先一覧に追加した大項目と中項目が表示されることを確認する。
9. 分類先一覧の `編集` を押し、大項目名、中項目名、アイコン、表示名、優先度、軸を変更して更新できることを確認する。
10. 分類先一覧の `削除` を押し、確認ダイアログでキャンセルした場合は削除されず、了承した場合は対象分類先が消えることを確認する。
11. サイト URL に `https://example.com/page` を入力して `分類先を追加` を押し、ドメインルールとして `ルール管理` に表示されることを確認する。
12. `全ルール削除` を押し、確認ダイアログでキャンセルした場合は削除されず、了承した場合は JSON が `[]` になり分類先一覧が空になることを確認する。
13. `初期ルールへ戻す` を押し、ゲーム、成人向け、削除予定の分類先が復元されることを確認する。
14. `候補作成` を押し、作成された候補の大項目と中項目がドロップダウンに表示されることを確認する。

期待結果:

- 指定された行構成でフォームが崩れない。
- 項目名と入力フォームが 2 行で揃い、項目名が改行されない。
- 必須項目と任意項目が視覚的に区別できる。
- 分類先設定の分類先は JSON ルールまたは候補作成結果から作られ、空 JSON だけでは初期分類先が混入しない。
- URL なしでは分類先ルール、URL ありではドメインルールを追加できる。
- 追加した分類先を編集、削除できる。
- 全ルール削除は確認後だけ実行され、初期ルール復元とは別操作として機能する。
- 追加ルールは保存前でもルール管理へ反映される。

## MT-4 JSON 編集と他モード同期

1. `JSON 編集` タブを開く。
2. 既存のルール JSON に新しい `domain` ルールを 1 件追加する。
3. JSON が有効な状態で `分類先設定`、`サイト割り当て`、`ルール管理` へ切り替える。
4. 追加した分類先やルールが各モードに反映されることを確認する。
5. サイト割り当てで、追加した分類先のルール数と該当ドメインの分類済み表示が更新されることを確認する。
6. JSON を `[]` にして `分類先設定` と `サイト割り当て` へ切り替え、初期分類先プリセットが表示されないことを確認する。
7. JSON を一時的に不正にして、他タブへ切り替えようとする。

期待結果:

- 有効な JSON は分類先設定、サイト割り当て、サイト割り当ての分類済み判定、ルール管理に同期される。
- 空 JSON の場合、候補作成で作った未追加候補がない限り分類先設定とサイト割り当ての分類先は空状態になる。
- 不正 JSON のまま他タブへ移動しようとするとエラーになり、古い有効状態が保持される。

## MT-5 サイト割り当て

1. `サイト割り当て` タブを開く。
2. `現在のブックマークを読み込む` を押す。
3. JSON ルールと候補が空の場合、左の分類先が空状態として表示されることを確認する。
4. `分類先設定` に戻って `候補作成` を実行し、再度 `サイト割り当て` へ戻る。
5. 左に候補作成由来の大項目と中項目、右にサイトカードが表示されることを確認する。
6. サイトカードに favicon または頭文字タイルが表示されることを確認する。
7. グリッド表示とリスト表示を切り替え、リスト表示では各サイトカードが同じ幅で縦に並ぶことを確認する。
8. サイトカードを分類先へドラッグ & ドロップする。
9. `分類済みを非表示` を有効にし、登録済みサイトが非表示になることを確認する。
10. 長いサイト名やサンプル名がカードからはみ出さず、省略表示されることを確認する。

期待結果:

- サイト割り当ては分類ルールだけを作成し、ブックマーク本体は移動しない。
- サイト割り当ての分類先は JSON ルールまたは候補作成結果と一致し、空 JSON だけで初期分類先が表示されない。
- 表示切替、分類済み非表示、長文省略、favicon 表示が崩れない。
- リスト表示のサイトカード幅と内部列幅が揃っている。

## MT-6 ルール管理

1. `ルール管理` タブを開く。
2. グループ、グリッド、リスト表示を切り替える。
3. 任意のルールの `削除` を押す。
4. 削除したルールが一覧から消えることを確認する。
5. `保存` を押してから Options を再読み込みし、削除状態が維持されることを確認する。

期待結果:

- 3 種類の表示が切り替わる。
- 削除したルールが初期補完で即時復活しない。
- 保存後も削除状態が維持される。

## MT-7 スキャンと分類プレビュー

1. Side Panel で `スキャンして分類` を押す。
2. ブックマーク件数、移動候補、作成予定が更新されることを確認する。
3. 分類結果にタイトル、ドメイン、移動先、分類理由が表示されることを確認する。
4. `整理 / 削除予定` に分類されるブックマークがある場合、分類結果では `削除予定`、変更プレビューでは `予定: 削除予定` と表示されることを確認する。
5. 変更プレビューに現在位置と予定位置または削除予定が表示されることを確認する。

期待結果:

- ルールに基づく大項目、中項目の整理先が表示される。
- `削除予定` 分類は移動先フォルダではなく削除アクションとして表示される。
- 未一致のものは `未分類 / ドメイン名` にフォールバックする。

## MT-8 リンク切れチェック

1. Side Panel で `リンク切れを確認` を押す。
2. 進捗が `確認 x/y 件` として更新されることを確認する。
3. HTTP 404/410 の候補がある場合、削除候補一覧に表示されることを確認する。
4. 通信エラー、認証エラー、403、429、5xx が削除候補になっていないことを確認する。
5. 削除候補がある場合だけ、内容を確認して `確認してリンク切れを削除` を押す。
6. 確認ダイアログでキャンセルした場合は削除されないことを確認する。

期待結果:

- HTTP 404/410 だけが削除候補になる。
- 削除は明示確認後だけ実行される。
- 削除後は再スキャンされ、プレビューから削除済みブックマークが外れる。

## MT-9 バックアップ、適用、復元

1. Side Panel で `現在状態をバックアップ` を押す。
2. 保存済み時刻と件数が表示されることを確認する。
3. `スキャンして分類` を実行し、変更プレビューを確認する。
4. `プレビュー済みの整理を適用` を押す。
5. 確認ダイアログで移動件数、削除予定件数、空フォルダ削除の説明を確認して実行する。
6. Chrome ブックマークバーで `分類済みブックマーク / 大項目 / 中項目` が作成され、ブックマークが移動していることを確認する。
7. 削除予定カテゴリに分類されたブックマークがある場合、整理適用後に削除されていることを確認する。
8. 空になった対象フォルダが削除されていることを確認する。
9. `直前の整理を復元` を押し、直前ジョブの範囲で移動済みブックマークが元へ戻ることを確認する。
10. `バックアップから復元` を押し、保存時点の親フォルダと順序へ戻ることを確認する。

期待結果:

- バックアップは繰り返しテストの復旧点として機能する。
- 整理適用は確認ダイアログを経由する。
- 削除予定カテゴリによる削除も整理適用確認を経由する。
- 空フォルダ削除は整理適用時に限定される。
- 直前復元とバックアップ復元のステータスが完了表示へ変わる。

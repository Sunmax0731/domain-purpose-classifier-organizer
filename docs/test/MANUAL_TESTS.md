# 手動テスト項目と確認手順

## 前提

- 作業ディレクトリ: `D:\AI\ChromeExtension\domain-purpose-classifier-organizer`
- 読み込み対象: `D:\AI\ChromeExtension\domain-purpose-classifier-organizer\extension`
- Chrome で `chrome://extensions` を開き、「デベロッパーモード」を有効にする。

## MT-1: 拡張機能を読み込める

1. `chrome://extensions` を開く。
2. 「パッケージ化されていない拡張機能を読み込む」を押す。
3. `D:\AI\ChromeExtension\domain-purpose-classifier-organizer\extension` を選ぶ。
4. 拡張機能名が `Bookmark Purpose Organizer` と表示されることを確認する。
5. エラーが表示されないことを確認する。

## MT-2: Popup から導線を確認できる

1. Chrome ツールバーの拡張機能アイコンを押す。
2. Popup に権限説明と導線が表示されることを確認する。
3. 「Side Panel を開く」を押し、Side Panel が表示されることを確認する。
4. 「Options を開く」を押し、ルール編集画面が開くことを確認する。

## MT-3: ブックマークをスキャンできる

1. Side Panel を開く。
2. 「スキャンして分類」を押す。
3. ブックマーク件数、分類件数、作成予定フォルダ数が表示されることを確認する。
4. 分類結果にタイトル、ドメイン、分類理由、移動先が表示されることを確認する。

## MT-3A: テスト前バックアップを作成できる

1. Side Panel を開く。
2. 「現在状態をバックアップ」を押す。
3. 確認ダイアログで了承する。
4. 「保存済み: 日時 / 件数」が表示されることを確認する。
5. 「バックアップから復元」ボタンが有効になることを確認する。

## MT-4: ルール編集が保存される

1. Options を開く。
2. 既存ルール JSON を編集し、任意の `keyword` ルールを 1 件追加する。
3. 「保存」を押す。
4. Side Panel に戻り「スキャンして分類」を押す。
5. 追加したキーワードに一致するブックマークが、指定した分類へ変わることを確認する。
6. 無効な JSON を入力した場合は保存されず、エラーが表示されることを確認する。

## MT-5: 差分プレビュー後にだけ整理を適用できる

1. Side Panel で分類結果と変更プレビューを確認する。
2. 「プレビュー済みの整理を適用」を押す。
3. 確認ダイアログが表示されることを確認する。
4. キャンセルした場合、ブックマークが移動しないことを確認する。
5. 了承した場合、作成予定フォルダが作られ、対象ブックマークが移動することを確認する。

## MT-6: 直前復元ができる

1. MT-5 で整理を適用する。
2. Side Panel の「直前の整理を復元」を押す。
3. 確認ダイアログで了承する。
4. 直前ジョブの対象ブックマークが元の親フォルダへ戻ることを確認する。
5. 復元できない項目がある場合、警告件数が表示されることを確認する。

## MT-6A: テスト用バックアップから復元できる

1. MT-3A でバックアップを作成する。
2. MT-5 で整理を適用し、ブックマークが移動したことを確認する。
3. Side Panel の「バックアップから復元」を押す。
4. 確認ダイアログで了承する。
5. バックアップ作成時点に存在したブックマークが、保存時点の親フォルダと順序へ戻ることを確認する。
6. 拡張機能が作成した空フォルダは自動削除されないことを確認する。
7. 復元できない項目がある場合、警告件数が表示されることを確認する。

## MT-7: 100 件以上のブックマークで表示を確認する

1. 100 件以上のブックマークがある Chrome プロファイルで Side Panel を開く。
2. 「スキャンして分類」を押す。
3. Side Panel のスクロール、分類結果、変更プレビューが操作不能にならないことを確認する。
4. 実行前に移動件数と作成予定フォルダ数を把握できることを確認する。

## MT-8: 権限説明と単一目的を確認する

1. `chrome://extensions` で拡張機能の詳細を開く。
2. 要求権限がブックマーク整理の目的に沿っていることを確認する。
3. README の権限説明と実際の `manifest.json` が一致することを確認する。
